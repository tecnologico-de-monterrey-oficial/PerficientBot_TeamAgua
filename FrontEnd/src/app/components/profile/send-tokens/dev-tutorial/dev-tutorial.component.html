Before you can access DevOps services using PerficientBot and interact with work items, you need to generate a personal access token. Follow the steps below:

  1. First, go to the Azure DevOps website and make sure you are logged in. Select the organization you want to work with from the left sidebar.
  2. In the top-right corner, click on the user settings icon and select Personal Access Tokens.
  3. You will be redirected to the personal access tokens page. Click on New Token to create a new token.
  4. Fill in the token form with the required information, such as the name and expiration date. For the scope, you can either choose full access (recommended) or select the specific permissions required for PerficientBot to work with work items, such as Work Items - Read, Write, & Manage.
  5. Click on Create and you will see a success message indicating that the token was created. Make sure to click on the copy symbol to copy the token to your clipboard. This will be your only chance to copy it.
  6. Now, go back to PerficientBot's My Services page and clear the content of the DevOps Token field. Paste the newly generated token into the field using Ctrl + V.
    7. That's it! You have successfully obtained your DevOps personal access token and can now access DevOps services and interact with work items using PerficientBot.
